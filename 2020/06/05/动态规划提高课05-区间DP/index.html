环形石子合并

原题链接

解题思路

最后一步操作肯定是把两堆石子合并，而这两堆肯定是对应原来的环形的两边互不相交的部分。所以相当于枚举环的断点，然后求数组的石子合并问题。这样的时间复杂度为O(n^4)

考虑破链成环。将原数组复制一份。那么在长度为2 * n 的区间上做区间DP，然后枚举所有的长度为n的情况，取最小和最大即可。

时间复杂度O(n^3)

  • 如果石子可以随便移，那就是哈弗曼编码问题
  • 如果不是环形，则直接区间DP即可

凸多边形的划分

原题链接

解题思路

定义状态dp[i][j]为从i - j 个定点围成的凸多边形的最小值。

那么在考虑dp[0][n - 1]的时候，因为0和n-1之间的边肯定属于某个三角形，那么就枚举这个三角形的另一个顶点。这个确定的三角形将这个多边形划分为了另外两部分互不相交的部分。然后枚举这个第三个定点即可。

$$dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j] + q[i]q[k]q[j] \quad for k > i && k < j)$$

转化为了区间DP的问题。

加分二叉树

原题链接

解题思路

给定中序遍历的结果，没棵子树在中序遍历中必然是连续的一段。在枚举所有可能的子树的时候，按照根节点的位置进行分类，这样两边的序列之间不会相互干扰。

定义状态dp[i][j]为i - j 的序列能够组成的树的得分的最大值。那么就有如下的状态转移方程：

$$dp[i][j] = max(dp[i][k-1] * dp[k+1][j] + q[k]) \quad for \quad k >= i \ && \ k <= j$$

注意边界情况：

  • 只有一颗子树，那么将空孩子看做1
  • 两颗子树都为空，那么两个空孩子看做0

上述方法能够得到最高的得分。最终需要输出中序遍历字典序最小的序列。

只需要记录下来i - j区间情况下，树的根是谁，然后从最开始的根开始回溯即可。

保证字典序最小的话，只需要在枚举根节点的时候，从小到大枚举，然后发现有相同的不更新即可。

如果从大到小枚举的话，发现有相同的值的话，需要更新。

棋盘分割

原题链接

解题思路

首先对最终的目标进行化简。

$$\sigma = \sqrt{\frac{\sum_{i=1}^{n}(x_i - \bar{x})^2}{n}}$$

$$\sum_{i = 1}^{n}(x_i - \bar{x})^2 = \sum_{i=1}^{n}x_i^2 - 2\bar{x_i}\sum_{i=1}^{n}x_i + n\bar{x}^2 = \sum_{i=1}^{n}x_i^2 - n\bar{x}^2$$

因为给定了输入数据和n,$\bar{x}$就是确定的，所以只需要让$\sum_{i=1}^{n}x_i^2$最小即可。

分析题意可知，决定一个问题的解有两个因素，一个是当前的局面，另一个是要切分的块数。如何描述当期的局面呢？可以使用矩形的左上角和右下角来进行描述。所以描述一个状态需要五维。

dp[x1][y1][x2][y2][k]表示将从(x1, y1)和(x2, y2)这个句型切分为k块，不同块的平方和的最小值。

那么在求解的时候，可以枚举第一刀的位置以及保留的是左右还是上下。这样就将问题转化为了规模更小的问题。

是用记忆化搜索的方式来进行处理。注意边界情况的处理。将所有元素初始化为正无穷，阻止非法的状态转移。

拿走瓶子

原题链接

解题思路

区间动态规划。在判断区间s[i, j]的时候，分为如下几种情况：

  • 如果a[i] == a[j]，那么最终可能i和j直接消除，那么次数就是dp[i+1][j-1]
  • 如果首尾元素不能够相互消除，那么因为只有相邻的才能够消除，可以枚举中间的分界点，然后将两侧消除的次数相加即可。
